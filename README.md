# TBN-IMCP4

IMC Prosperity 4 algorithmic trading competition entry.

## Prerequisites

- Python 3.11+
- Node.js 18+ — [nodejs.org](https://nodejs.org)
- pnpm — `npm install -g pnpm`
- [`prosperity4btx`](https://pypi.org/project/prosperity4btx/) backtester — `pip install prosperity4btx`

## First-time setup

```bash
git clone --recurse-submodules <repo-url>
cd TBN-IMCP4
python setup_visualizer.py   # installs and builds the local visualizer (once)
```

If you already cloned without `--recurse-submodules`:
```bash
git submodule update --init --recursive
python setup_visualizer.py
```

## Project Structure

```
TBN-IMCP4/

```

## Running a Backtest

```bash
python run_backtest.py 0
```

This will:
1. Run the backtest for day `0` → saves `output.log`
2. Start a local server on port `8765` serving both `output.log` and the visualizer
3. Open `http://localhost:8765/imc-prosperity-3-visualizer/?open=http://localhost:8765/output.log` in your browser

Press `Ctrl+C` to stop the server.

**Options:**

```bash
python run_backtest.py 0              # day 0 (tutorial round)
python run_backtest.py 1              # day 1
python run_backtest.py 0 --merge-pnl  # merge P&L across days
```

### Manual (no visualizer)

```bash
prosperity4btx message.py 0 --out output.log
```

## IMC Submission vs. Logger Boilerplate

`message.py` is submitted to the IMC platform as-is — the Logger class and `Trader` class live in the same file.

### What the IMC platform template gives you

The platform provides this stub as a starting point:

```python
logger = Logger()

class Trader:
    def run(self, state: TradingState) -> tuple[dict[Symbol, list[Order]], int, str]:
        result = {}
        conversions = 0
        trader_data = ""   # plain empty string

        # TODO: Add logic

        logger.flush(state, result, conversions, trader_data)
        return result, conversions, trader_data
```

### Key differences in this implementation

| | IMC template | This repo (`message.py`) |
|---|---|---|
| `trader_data` type | `str` (empty string, unused) | `dict` loaded from JSON, serialized back at the end |
| `Trader.run()` body | stub (`# TODO`) | full EMERALDS + TOMATOES strategies |
| `logger.flush()` argument | `trader_data = ""` | `traderData = json.dumps(trader_data)` (serialized dict) |

### Rules for the Logger class

- **Do not modify** the `Logger` class. It is required verbatim for the visualizer to parse the output.
- **Always call** `logger.flush(state, result, conversions, traderData)` as the last statement before `return`. Skipping it produces empty logs.
- **Use `logger.print()`** instead of `print()` for any debug output. Bare `print()` calls bypass the logger and will break the log format.
- The `Logger` class must be present in the submitted file — it is not provided by the platform at runtime.

## Algorithm (`message.py`)

Two strategies running in parallel:

| Product | Strategy | Logic |
|---|---|---|
| EMERALDS | Market Making | Post limit orders 1 tick inside the spread; 15-unit chunks |
| TOMATOES | Mean Reversion | Buy when ask < 20-period mean − 10; sell when bid > mean + 10 |

Position limit is **80 units** per product in either direction.

### Debug Logging

Use `logger.print()` instead of `print()` anywhere in `Trader.run()`. These messages appear in the visualizer's **Lambda Logs** panel.

```python
logger.print(f"TOMATOES mean={mean_price:.1f} bid={best_bid} ask={best_ask}")
```

### Adding a New Product

Add an `elif product == "YOUR_PRODUCT":` branch in the `for product in state.order_depths` loop. Follow the existing pattern: read order depth, compute signal, build `orders` list, respect the position limit.

## Visualizer

The visualizer (`visualizer/`) is a git submodule pointing to [jmerle/imc-prosperity-3-visualizer](https://github.com/jmerle/imc-prosperity-3-visualizer). It is built locally by `setup_visualizer.py` and served entirely from Python — no internet connection required at runtime.

`run_backtest.py` routes a single server on port `8765`:
- `/imc-prosperity-3-visualizer/` → `visualizer/dist/`
- `/output.log` → project root `output.log`
